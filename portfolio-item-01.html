Basic web application Online Book DB Interface

C#, ASP.NET, JavaScript

I began this project without any prior knowledge of the C# language and ASP.NET framework, outside of a simple "hello world" console application. I learned an incredible amount about the framework by the time I achieved the results I have here. I imagine I have only grazed the surface of the potentially great features that are included in the most recent update. I am still steadily making changes to this application, discovering new techniques and interactions between the user and the server.

This application was initially constructed using Razor Pages in order to properly add functionality. This is different than the MVC approach, but effectively achieves a similar end result in this scenario. I am working on an MVC version of this very application and will have this completed in the next week or so. The basic web application creates a single database table "Book" with several attributes. I have included several functions within this application in order to allow the user to interact with the database table from the web application. Initial features included "Add New Entry", "Delete", and "Edit". The most recent upload includes a feature "Upsert", essentially combining the "Add New Entry" and "Edit" features into the same functional code.

Both of the tables displayed are identical, but I've utilized two different techniques here. The first table uses simple queries to call the data from the database, requiring a page refresh to display any new changes to the data. The second table uses a combination of JavaScript and API calls to perpetually call the data from the database, providing the user a seamless interactive experience with the data. Any new entries/updates/deletes should be displayed on the table instantly.
